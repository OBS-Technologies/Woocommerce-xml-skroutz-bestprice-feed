OBS WP Woocommerce XML Feed για Skroutz.gr και bestprice.gr

Εγκατάσταση και παραμετροποίηση Plugin

Τελευταία ενημέρωση: Μαρ 25, 2017

Εγκατάσταση

1ο Βήμα

Εντοπίζουμε το αρχείο .zip που κατεβάσαμε με τίτλο obs_custom_feeds.

Click to zoom

2o Βήμα

Στον πίνακα διαχείρισης του WordPress κάνουμε κλικ στην επιλογή "Πρόσθετα (Plugins)" -> "Προσθήκη Νέου (Add New)".

Click to zoom

3ο Βήμα

Στο εμφανιζόμενο παράθυρο επιλέγουμε "Μεταφόρτωση Προσθέτου (Upload PLugin)" -> "Επιλογή αρχείου (Browse.. / Choose File..)" - > "Εγκατάσταση Τώρα (Install Now)"

Click to zoom

4ο Βήμα

Τέλος, επιλέγουμε "Ενεργοποίηση Προσθέτου (Activate Plugin)"

Click to zoom

5ο Βήμα

Εφόσον τα βήματα έχουν εκτελεστεί σωστά, τότε στον πίνακα προσθέτων θα υπάρχει το πρόσθετο (Plugin) OBS WooCommerce Custom Feeds ενεργοποιημένο :)

Click to zoom

Παραμετροποίηση Plugin

Αριστερά στην λίστα εργαλείων θα εμφανιστεί μία καινούργια επιλογή με το όνομα Custom Feeds με τις παρακάτω επιλογές:

  • Custom Feeds
  • Messages
  • Settings
  • Update Feeds

Παρακάτω αναλύεται η χρήση της κάθε επιλογής

Custom Feeds

Click to zoom


Η καρτέλα Custom Feeds χωρίζεται σε δύο μέρη:

  • Στην ενότητα παραμετροποίησης υπαρχόντων πεδίων
  • Στην ενότητα προσθήκης νέου πεδίου
Προσθήκη νέου πεδίου και κατανόησή τους

Για την δημιουργία νέου πεδίου που θέλουμε να συμπεριλαμβάνεται στα Feeds του Skroutz και του Best Price πρέπει να ρυθμίσουμε 5 απλές επιλογές.

  • WC Label

    Στο συγκεκριμένο πεδίο ορίζεται τό όνομα του πεδίου όπως έχει οριστεί εντός του WooCommerce στο αντίστοιχο Custom Field ή Attribite.

    Παράδειγμα 1 με Custom Field: Αν ορίσουμε ένα Custom Field με το όνομα ISBN (δεδομένου ότι τα custom fields ορίζονται με λατινικούς χαρακτήρες) στο WC Label θα συμπληρώσουμε ISBN.

    Το όνομα όπως εμφανίζεται στο custom field του προϊόντος

    Στο WC Label γράφουμε ISBN (το WC Field Type και Feed Tag κτλ. αναλύονται παρακάτω)

    Click to zoom

    Παράδειγμα 2 με Attribute: Αν ορίσουμε ένα Attribute με το όνομα Χρώμα και slug color(δεδομένου ότι τα slugs ορίζονται με λατινικούς χαρακτήρες) στο WC Label θα συμπληρώσουμε pa_color.

    Προσοχή:

    Είναι σημαντική η προσθήκη του προθέματος pa_ μπροστά από το όνομα του slug (χωρίς κενό), όταν το συμπληρώνουμε στο WC Label!

    Το slug όπως εμφανίζεται στο σχετικό attribute

    Παράδειγμα 3 με Product Description:
    Δεν χρειάζεται να χρησιμοποιηθεί αυτή η επιλογή και υπάρχει προληπτικά σε περίπτωση που ένας από τους δύο παρόχους αλλάξει τον τρόπο που δέχεται την Περιγραφή Προϊόντος - Product Description.

    Σε αντίθεση με τις παραπάνω επιλογές το Product Description έχει προληπτικό ρόλο. Υπάρχει ήδη στα προκαθορισμένα πεδία του Plugin (που βρίσκονται διαθέσιμα για επεξεργασία) το πεδίο Product Description και έχει οριστεί με βάση την παρούσα λειτουργεία των Skroutz και BestPrice.

  • WC Field Type

    Υπάρχουν 3 επιλογές για το WC Field Type


    • Custom Field: Εδώ ορίζεται ότι θα χρησιμοποιηθεί Custom Field για το feed. Ta custom fields δημιουργούνται για τιμές που έχουν μοναδικότητα όπως: Serial Numbers, ISBNS, Part Numbers κτλ.
    • Attibute: Εδώ ορίζεται ότι θα χρησιμοποιηθεί Attribute για το feed. Ta attributes δημιουργούνται για τιμές που έχουν επανάληψη και κατηγοριοποιούν προϊόντα όπως: Κατασκευαστή, χρώμα, μέγεθος.
    • Product Description: Εδώ ορίζεται ότι θα χρησιμοποιηθεί Product Description field για το feed. Ta Product descriptions μεταφέρουν την περιγραφή του προϊόντος στο εκάστοτε Skroutz ή BestPrice
  • Feed Tag

    Στο Feed Tag συμπληρώνουμε το XML Tag που δέχεται ο εκάστοτε πάροχος για να διαχειριστεί το αντίστοιχο tag.
    Την παρούσα στιγμή BestPrice και Skroutz χρησιμοποιούν τα ίδια Feed Tags
    Δείτε σχετικές λεπτομέρειες στα:

    • Skroutz for Developers
    • BestPrice.gr XML Specifications PDF

    Παράδειγμα:
    Τα XML δέχονται για Attribute χρώματος το εξής XML Tag <color> Blue </color> οπότε

    Συμπληρώνουμε στο πεδίο Feed Tag αντίστοιχα το όνομα του XML Tag (color σε αυτή την περίπτωση)

    Click to zoom
  • Disable Variations

    Επιλέγοντας αυτό το πεδίο καθορίζουμε αν στο XML Feed που θα παρέχουμε, αν επιθυμούμε να περαστούν και οι παραλλαγές του προϊόντως στους καταλόγους Skroutz και BestPrice.

    • Αν είναι επιλεγμένο, τότε θα αγνοηθούν οι παραλλαγές στο XML Feed που θα παραγει το Plugin και θα διατήθεται μόνο η προκαθορισμένη του παραλλαγή
    • Αν δεν είναι επιλεγμένο, τότε όλες οι παραλλαγές του προϊόντος θα περαστούν σαν μοναδικές καταχωρήσεις-προϊόντα στο XML Feed
  • Use in Feeds

    Στα συγκεκριμένα checkboxes επιλέγουμε σε ποιον πάροχο προωθούμε αυτό το Field (πχ. σε περίπτωση που το χρησιμοποιεί μοναδικά ένας πάροχος κάποιο ιδιαίτερο tag ή επιθυμούμυ εμείς να ενημερώνουμε το XML του ενός για μία συγκεκριμένη πληροφορία)

Messages

Στην περιοχή messages υπάρχουν δύο επιλογές (όπως φαίνεται στην παρακάτω εικόνα). Αφορούν τα μηνύματα που περνάμε για προϊόντα που δεν έχουν άμεση διαθεσιμότητα.

  • Στο πεδίο Backorder καταχωρούμε το μήνυμα που θέλουμε να εμφανίζεται όταν ένα προϊόν το φέρνουμε κατόπιν παραγγελίας.
  • Στο πεδίο Out of Stock καταχωρούμε το μήνυμα που θέλουμε να εμφανίζεται όταν ένα προϊόν έχει εξαντληθεί και θέλουμε να ενημερώσουμε τον πελάτη για έναν εκτιμόμενο χρόνο παράδοσης.

Κατόπιν παραμετροποίησής τους πατάμε Save Changes

Settings

Στην περιοχή settings υπάρχουν τέσσερις επιλογές που πρέπει να ορίσουμε (όπως φαίνεται στην παρακάτω εικόνα):


  • Get Product Image From: Σε αυτήν την επιλογή δηλώνουμε αν θέλουμε το XML να παρέχει link της εικόνας του προϊόντος από την Προβαλλόμενη Εικόνα (Featured Image) ή από την Product Gallery του WooCommerce.
  • Default Product Image URL:Εδώ καταχωρούμε (εφόσον υπάρχει) τον σύνδεσμο της εικόνας που θα συνοδεύει το xml, για το προϊόντα που δέν διαθέτουν εικόνες
  • Include Product Categories: Εδώ έχουμε την δυνατότητα πολλαπλής επιλογής κατηγοριών προϊόντων που θέλουμε να περιλαμβάνονται στο XML ή εναλλακτικά επίλέγουμε το --ALL-- για να περιλαμβάνονται όλες οι κατηγορίες στο XML Feed
  • Feed Generation:
    • Manual: Αν θέλουμε να στέλνουμε το ανανεωμένο XML κατά βούληση (δηλαδή όταν έχουμε ολοκληρώσει ένα σύνολο αλλαγών) χρησιμοποιούμε την επιλογή Manual.
      Κατ' επέκταση για να αποσταλεί το Feed θα πάμε στην καρτέλα Update Feed και θα πατήσουμε Run update proccess
    • Automatic: Αν θέλουμε το XML Feed να ανανεώνεται αυτόματα (on Request)

Feed Generation Server Stress

Ανάλογα με τον αριθμό των προϊόντων ενός e-shop καθορίζονται οι πόροι (δηλαδή η ταχύτητα) που μπορεί ο server να παράγει ένα XML αρχείο. Σε έναν μεγάλο όγκο προϊόντων αποφεύγουμε το Automatic Feed Generation γιατί μπορεί να δημιουργήσει μεγάλες καθυστερίσεις στην λειτουργία της ιστοσελίδας. Σε αυτές τις περιπτώσεις επιλέγουμε Manual Feed Generation και πραγματοποιούμε το Update proccess κατά βούληση (ιδανικα ώρες που το site δεν έχει υψηλή επισκεψιμότητα)

Τέλος πατάμε Save Settings

Update Feeds

Την συγκεκριμένη καρτέλα την επισκεπτόμαστε μόνο όταν θέλουμενα παράγουμε το ανανεωμένο Feed Προϊόντων σε Skroutz και BestPrice.
Πατάμε Run update proccess και το ανανεομένο feed δημιουργείται
